# Task 4 — System Design (Written, No Code)

No code here. We want to see how you think before building. Keep it short — bullet points and simple diagrams are fine. Half a page to one page total.

Answer these in a file called `DESIGN.md`:

## 1. Graspia at scale

Graspia will serve many businesses at once. Each business has its own products, tone, and rules.

- How would you store each business's knowledge so the AI answers correctly for the right business?
- How do you stop one business's data leaking into another's conversations?
- How do you keep AI costs under control as conversations grow?

## 2. Reliability

- What happens if the AI API is down or slow during a live client chat? What does the client see?
- How do you make sure no lead or message is ever lost?

## 3. Human handover in the real world

- A human is busy and doesn't pick up a handover for 10 minutes. What should happen?
- How do you track which conversations need a human and which are handled?

## 4. Your first 2 weeks

If we hired you, what would you build or fix first across these two projects, and why?

---

## What we check

- Clear thinking over fancy words.
- Do you think about cost, failure, and data safety — not just the happy path?
- Practical priorities, not theory.
